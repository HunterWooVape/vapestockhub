# Appendix B - 单位模型

## 1. 本页定位

本页的作用不是给出一套“看起来很漂亮”的财务数字，而是建立一套与当前阶段真实业务形态一致的单位经济逻辑。

本页要解决的问题是：

- 我们到底以什么为核心经济单位
- 成本发生在哪一层
- 价值产生在哪一层
- 为什么这个模型会随着执行优化而变得更强

---

## 2. 核心结论

```text
Our core unit is not a completed transaction.
Our core unit is a qualified inquiry generated from structured inventory.
```

中文：

```text
我们的核心单位，不是已完成交易，
而是由结构化库存持续产生的高质量询盘。
```

---

## 3. 最关键的认知升级

```text
Liquidity is the first economic output.
Transactions are a later monetization layer.
```

中文：

```text
流动性，是最早的经济产出；
交易，是更后层的变现结果。
```

这句话很重要，因为它定义了你当前阶段的经济逻辑。

---

## 4. 为什么不能用传统电商单位模型

如果直接用传统电商口径去讲：

- 客单价
- GMV
- 订单数
- 成交佣金率

会有两个问题：

- 不符合当前阶段真实业务
- 会把项目错误讲成完整交易平台

因此，单位模型必须围绕：

- `有效结构化库存`
- `有效询盘`

来建立。

---

## 5. 建议采用双单位模型

### 5.1 Unit A - Active Structured Listing

定义：

- 一条被收集、标准化、复核并成功上线的有效库存

意义：

- 它是供给侧生产单位
- 它代表“碎片化库存是否已被转化为可流通供给”

### 5.2 Unit B - Qualified Inquiry

定义：

- 一个通过页面或私域分发而产生，且具备真实采购意向与有效联系方式的询盘

意义：

- 它是需求侧价值单位
- 它代表“结构化供给是否真的被转化为商业意图”

---

## 6. 经济逻辑链条

建议统一使用以下逻辑链条：

```text
Fragmented inventory
-> Active structured listing
-> Searchable / distributed visibility
-> Buyer contact
-> Qualified inquiry
-> Monetizable value
```

中文：

```text
碎片化库存
-> 有效结构化库存
-> 搜索化 / 分发可见性
-> 买家联系
-> 有效询盘
-> 可变现价值
```

---

## 7. 供给侧成本定义

### 单条有效结构化库存成本

```text
Cost per Active Structured Listing
= supply intake cost
+ cleaning / standardization cost
+ manual review cost
+ media handling cost
+ publishing cost
```

中文：

```text
单条有效结构化库存成本
= 供给接入成本
+ 数据清洗标准化成本
+ 人工复核成本
+ 图片处理成本
+ 上架发布成本
```

### 这一层的核心问题

- 一条库存从“原始资料”变成“可展示供给”要花多少成本
- 单人每天能处理多少条
- 随着流程标准化，人效能否提高

---

## 8. 需求侧成本定义

### 单个有效询盘成本

```text
Cost per Qualified Inquiry
= (total listing production cost + distribution cost)
/ total qualified inquiries generated
```

中文：

```text
单个有效询盘成本
= （库存生产总成本 + 分发成本）
/ 有效询盘总数
```

### 这一层的核心问题

- 每 100 条库存能产出多少有效询盘
- 哪些页面 / 市场 / 品牌询盘密度更高
- 分发成本与询盘质量之间如何平衡

---

## 9. 单个有效询盘价值

建议用“价值框架”来表达，而不是过早写死成交数字。

```text
Value per Qualified Inquiry
= direct monetization value
+ expected match value
+ future seller-side monetization value
```

中文：

```text
单个有效询盘价值
= 当前直接变现价值
+ 撮合预期价值
+ 后续卖家侧变现隐含价值
```

### 价值来源可以包括

- 当前线索服务价值
- 当前撮合服务价值
- 卖家侧优先分发或曝光价值
- 未来会员、数据或交易支持的隐含价值

---

## 10. 单位贡献表达

```text
Contribution per Qualified Inquiry
= value per qualified inquiry - cost per qualified inquiry
```

中文：

```text
单个有效询盘贡献
= 单个有效询盘价值 - 单个有效询盘成本
```

---

## 11. 建议追踪的 5 个核心指标

### 11.1 Cost per Active Structured Listing

- 单条有效结构化库存成本

### 11.2 Listings per Operator per Day

- 单人单日可处理库存数

### 11.3 Qualified Inquiries per 100 Listings

- 每 100 条库存带来的有效询盘数

### 11.4 Cost per Qualified Inquiry

- 单个有效询盘成本

### 11.5 Response-to-Commercial-Opportunity Rate

- 有效询盘转化为可跟进商机的比例

---

## 12. 单位经济改善逻辑

```text
Operational learning improves unit economics over time.
```

中文：

```text
运营学习曲线，会持续改善单位经济模型。
```

建议强调以下改善方向：

- 单条库存处理成本下降
- 页面到联系转化率提升
- 单位库存询盘产出提升
- 高意图需求承接效率提升
- 低质量库存占比下降

### 英文版

```text
Unit Economics

VapeStockHub’s unit economics are built around liquidity creation, not completed transactions.

We use a dual-unit model:
1. Active Structured Listing
2. Qualified Inquiry

Cost is incurred to turn fragmented inventory into active listings.
Value is created when active listings consistently generate qualified inquiries.

Over time, unit economics improve through lower listing production cost, higher inquiry yield, and better conversion efficiency.
```

### 中文版

```text
单位模型

VapeStockHub 的单位经济模型，建立在流动性创造之上，而不是建立在已完成交易之上。

```

我们的单位模型不是围绕成交订单设计的，而是围绕库存流动性设计的。当前阶段，我们定义两个核心单位：第一是有效结构化库存，也就是一条被收集、标准化、复核并上线的库存；第二是有效询盘，也就是由这些库存被发现并转化出来的真实买家意图。成本主要发生在把碎片化库存转化为标准化供给的过程中，价值主要产生于这些供给能否持续带来高质量询盘。随着流程优化、页面效率提升和高意图供给筛选能力增强，单条库存成本会下降，单个有效询盘产出会提高，整个单位经济会越来越成立。

### 英文版

Our unit economics are not built around completed transactions. They are built around liquidity creation. At this stage, we use two core units: the active structured listing and the qualified inquiry. Cost is incurred when fragmented inventory is collected, standardized, reviewed, and published. ```

## 18. 这一页常见错误

- 直接套电商 GMV 模型
- 把成交佣金写成当前阶段的唯一经济核心
- 编造过于精确、无法验证的漂亮数字
- 只讲询盘数量，不讲询盘质量
- 只讲收入，不讲单条库存成本

---

## 19. 统一金句

建议优先保留：

1. `Our core unit is a qualified inquiry generated from structured inventory.`
2. `Liquidity is the first economic output.`
3. `We optimize inquiry economics before transaction economics.`
4. `Better supply selection improves both conversion and margin.`
